Markdown 完整教學指南

AI 時代的必備書寫語言 —— 從零基礎到進階應用，一篇搞定。

什麼是 Markdown？

Markdown 是一種輕量級標記語言，由 John Gruber 於 2004 年創造。 它讓你用純文字就能寫出排版精美的文件，不需要像 Word 那樣點來點去。

簡單來說：你只需要記住幾個符號，就能產生標題、粗體、列表、連結、表格等格式。

為什麼在 AI 時代一定要學 Markdown？

🤖

AI 的通用語言

ChatGPT、Claude、Gemini 等 AI 工具的輸入與輸出幾乎都使用 Markdown 格式。學會 Markdown，你就能更精準地和 AI 溝通。

📝

Prompt 工程必備

撰寫結構化的 Prompt 時，使用 Markdown 的標題、列表、程式碼區塊能讓 AI 更好地理解你的意圖，產出更高品質的回覆。

💻

開發者的日常

GitHub README、技術文件、API 文件、部落格文章 —— 全都用 Markdown。學一次，到處都能用。

⚡

極速書寫

不需要離開鍵盤去按工具列按鈕。用幾個簡單的符號，打字速度就是排版速度。

1. 標題（Headings）

使用 # 號來建立標題，# 的數量決定標題的層級（1 到 6 級）。

語法

# 一級標題（H1）
## 二級標題（H2）
### 三級標題（H3）
#### 四級標題（H4）
##### 五級標題（H5）
###### 六級標題（H6）

注意事項

  • # 後面必須加一個空格，否則不會生效
  • 一篇文章通常只用一個 H1 作為主標題
  • 層級不要跳著用（例如 H1 之後直接用 H4）

AI 時代技巧

在 Prompt 中使用標題來組織指令結構，能幫助 AI 更好地理解你的需求：

# 角色設定
你是一位資深的 Python 開發者。

## 任務
幫我重構以下程式碼。

## 要求
- 提高可讀性
- 加入錯誤處理
- 加入型別註解

2. 文字格式（Text Formatting）

粗體

用兩個星號 ** 或兩個底線 __ 包圍文字：

**這是粗體文字**
__這也是粗體文字__

斜體

用一個星號 * 或一個底線 _ 包圍文字：

*這是斜體文字*
_這也是斜體文字_

粗斜體

用三個星號 *** 包圍文字：

***這是粗斜體文字***

刪除線

用兩個波浪號 ~~ 包圍文字：

~~這段文字被刪除了~~

螢光標記（部分平台支援）

用兩個等號 == 包圍文字：

==這是螢光標記的文字==

上標與下標（部分平台支援）

H~2~O（下標）
X^2^（上標）

3. 段落與換行（Paragraphs & Line Breaks）

段落

用一個空行來分隔段落：

這是第一段。

這是第二段。中間隔了一行空行。

強制換行

在行尾加上兩個空格然後按 Enter，或者使用 <br> 標籤：

這是第一行（行尾有兩個空格）
這是第二行

或者用 HTML 標籤：
第一行<br>第二行

常見錯誤：直接按一次 Enter 不會產生換行，文字會被連在同一行。

4. 列表（Lists）

無序列表

使用 -、* 或 + 作為項目符號：

- 蘋果
- 香蕉
- 橘子

* 項目一
* 項目二

+ 項目 A
+ 項目 B

有序列表

使用數字加句點：

1. 第一步
2. 第二步
3. 第三步

小技巧：其實數字不需要連續，Markdown 會自動排序。但為了可讀性，建議按順序寫。

巢狀列表

用縮排（2 或 4 個空格）來建立子列表：

- 水果
  - 蘋果
    - 富士蘋果
    - 青蘋果
  - 香蕉
- 蔬菜
  - 花椰菜
  - 胡蘿蔔

混合列表

1. 第一步
   - 注意事項 A
   - 注意事項 B
2. 第二步
   - 注意事項 C

5. 連結（Links）

行內連結

[顯示文字](https://example.com)

[Google](https://www.google.com "滑鼠懸停時顯示的提示文字")

參考式連結

適合同一個連結被多次引用的情況：

這是 [Markdown 官方文件][md-docs] 的連結。

你也可以參考 [Markdown 指南][md-docs]。

[md-docs]: https://www.markdownguide.org "Markdown Guide"

自動連結

<https://www.google.com>
<email@example.com>

頁內錨點連結

連結到同一頁面的標題：

[跳到標題區段](#標題的英文小寫-用連字號連接)

6. 圖片（Images）

基本語法

和連結很像，前面多一個驚嘆號 !：

![替代文字](圖片網址)

![可愛的貓咪](https://example.com/cat.jpg "貓咪照片")

參考式圖片

![替代文字][img-id]

[img-id]: https://example.com/image.png "圖片標題"

圖片加連結

把圖片語法包在連結語法裡面：

[![替代文字](圖片網址)](連結網址)

調整圖片大小（使用 HTML）

<img src="image.png" alt="說明" width="300">

7. 程式碼（Code）

行內程式碼

用單個反引號 ` 包圍：

請使用 `print("Hello")` 來輸出文字。

變數 `x` 的值是 `42`。

程式碼區塊

用三個反引號 ``` 包圍，可以指定程式語言以啟用語法高亮：

```python
def greet(name):
    """和某人打招呼"""
    return f"你好，{name}！歡迎來到 AI 的世界。"

print(greet("小明"))
```

8. 引用（Blockquotes）

基本引用

用 > 開頭：

> 學習 Markdown 最好的時間是十年前，其次是現在。

多行引用

> 這是第一行引用。
>
> 這是第二段引用。
> 可以跨越多行。

巢狀引用

> 這是外層引用。
>
> > 這是內層引用。
>
> 回到外層。

引用中使用其他格式

> **注意：** 引用裡面可以使用 *斜體*、`程式碼`、列表等。
>
> - 項目一
> - 項目二

GitHub 風格的提示區塊

GitHub 和部分平台支援特殊的提示語法：

> [!NOTE]
> 這是一個備註提示。

> [!TIP]
> 這是一個小技巧。

> [!WARNING]
> 這是一個警告。

> [!CAUTION]
> 這是一個危險警告。

> [!IMPORTANT]
> 這是一個重要提示。

9. 表格（Tables）

基本表格

用 | 分隔欄位，用 --- 分隔標題和內容：

| 功能     | Markdown 語法  | 說明           |
| -------- | -------------- | -------------- |
| 粗體     | `**文字**`     | 兩個星號包圍   |
| 斜體     | `*文字*`       | 一個星號包圍   |
| 程式碼   | `` `code` ``   | 反引號包圍     |
| 連結     | `[文字](URL)`  | 方括號加圓括號 |

對齊方式

在分隔線中使用冒號 : 來控制對齊：

| 靠左對齊     | 置中對齊      | 靠右對齊     |
| :----------- | :-----------: | -----------: |
| 左           | 中            | 右           |
| Apple        | Banana        | $1.00        |
| Strawberry   | Orange        | $2.50        |

注意事項

  • 表格的欄位不需要完全對齊（但對齊比較好讀）
  • 表格中可以使用連結、粗體、斜體、行內程式碼
  • 不支援表格中的換行（需使用 <br> 標籤）
  • 不支援合併儲存格（需使用 HTML）

10. 分隔線（Horizontal Rules）

用三個以上的星號、減號或底線：

***

---

___

三種寫法效果完全相同，建議統一使用 ---。

11. 任務清單（Task Lists）

GitHub Flavored Markdown (GFM) 支援任務清單，非常適合追蹤待辦事項：

- [x] 學會 Markdown 基本語法
- [x] 了解標題和文字格式
- [ ] 練習表格語法
- [ ] 學會在 AI Prompt 中使用 Markdown
- [ ] 寫一篇自己的 Markdown 文件

AI 時代技巧

用任務清單給 AI 佈置多個子任務：

請幫我完成以下任務：

- [ ] 檢查程式碼中的安全漏洞
- [ ] 為每個函式加上文件字串
- [ ] 撰寫單元測試
- [ ] 產生 API 文件

12. 註腳（Footnotes）

部分 Markdown 引擎支援註腳語法：

Markdown 是由 John Gruber 發明的[^1]。

它的設計哲學是讓文件「原始碼」也具有可讀性[^readability]。

[^1]: John Gruber 在 2004 年發布了 Markdown。
[^readability]: 這是 Markdown 最重要的設計目標之一。

13. 跳脫字元（Escaping）

如果你要顯示 Markdown 使用的特殊符號本身，在前面加上反斜線 \：

\* 這不是斜體 \*
\# 這不是標題
\[這不是連結\](也不是)
\`這不是程式碼\`

可以跳脫的字元

\   反斜線
`   反引號
*   星號
_   底線
{}  大括號
[]  方括號
()  圓括號
#   井號
+   加號
-   減號
.   句點
!   驚嘆號
|   管線符號

14. 在 Markdown 中使用 HTML

大多數 Markdown 引擎允許你直接嵌入 HTML：

常見用法

<!-- 這是 HTML 註解，不會被渲染 -->

<details>
  <summary>點擊展開更多內容</summary>
  這裡是隱藏的內容，點擊上面的文字就會顯示。
</details>

<kbd>Ctrl</kbd> + <kbd>C</kbd> 複製

<mark>螢光標記文字</mark>

<div align="center">
  置中的內容
</div>

注意事項

  • HTML 區塊內的 Markdown 語法通常不會被解析
  • 有些平台（如 GitHub）會過濾部分 HTML 標籤（如 <script>、<style>）
  • 能用 Markdown 解決的就不要用 HTML，保持簡潔

15. AI Prompt 中的 Markdown 技巧

在 AI 時代，Markdown 已成為人機溝通的「共同語言」。以下是在 Prompt 中善用 Markdown 的技巧：

結構化 Prompt 模板

# 角色
你是一位 [專業領域] 的專家。

## 背景
[提供相關的背景資訊]

## 任務
[清楚描述你要 AI 做什麼]

## 要求
- 要求一
- 要求二
- 要求三

## 輸出格式
[描述你期望的輸出格式]

## 範例
[提供一個輸入/輸出的範例]

用分隔線分離上下文

以下是需要翻譯的文章：

---

[文章內容]

---

請將上面的文章翻譯成繁體中文，保持原有的 Markdown 格式。

用程式碼區塊包裹數據

請分析以下 JSON 數據，找出異常值：

```json
{
  "users": [
    {"name": "Alice", "age": 25},
    {"name": "Bob", "age": -3},
    {"name": "Charlie", "age": 200}
  ]
}
```

用表格提供結構化輸入

請根據以下產品資訊，撰寫行銷文案：

| 產品名稱 | 價格   | 特色功能       |
| -------- | ------ | -------------- |
| AI 助手  | $9/月  | 語音對話、翻譯 |
| 智能筆記 | $5/月  | 自動摘要、整理 |

用任務清單追蹤多步驟任務

我正在建一個網站，請幫我逐步完成：

- [x] 設計首頁 wireframe
- [ ] 建立導覽列
- [ ] 實作 RWD 響應式設計
- [ ] 加入深色模式

CLAUDE.md / AGENTS.md / RULES.md

現代 AI 編程工具（如 Claude Code、Cursor、GitHub Copilot）都支援用 Markdown 檔案來設定 AI 的行為規範。這些檔案就是用 Markdown 撰寫的：

# CLAUDE.md 範例

## 程式碼規範
- 使用 TypeScript，不要用 JavaScript
- 優先使用函式式程式設計風格
- 所有函式都要加型別註解

## 測試規範
- 每個功能都要有單元測試
- 測試覆蓋率不低於 80%

## Git 規範
- Commit message 使用英文
- 遵循 Conventional Commits 格式

16. Markdown 工具推薦

編輯器

📓

Obsidian

最佳 Markdown 筆記工具，支援雙向連結、圖譜視圖、豐富的外掛生態。完全免費。

💜

VS Code

微軟的免費編輯器，內建 Markdown 預覽，搭配外掛功能強大。程式設計師首選。

✍️

Typora

所見即所得的 Markdown 編輯器，打字的同時即時渲染。介面簡潔優雅。

線上工具

🌐

HackMD / CodiMD

線上多人協作 Markdown 編輯器，支援即時同步、投影片模式、書本模式。

📄

Notion

雖然有自己的格式系統，但支援 Markdown 快捷鍵輸入，匯出也支援 Markdown。

AI 工具中的 Markdown

  • ChatGPT / Claude：輸入和輸出都原生支援 Markdown
  • Claude Code：整個工作流程都基於 Markdown（CLAUDE.md 設定檔）
  • Cursor / Windsurf：AI 程式碼編輯器，Rules 設定使用 Markdown
  • GitHub Copilot：支援 Markdown 文件的自動補全

總結

Markdown 是 AI 時代最值得投資學習的書寫語言之一。它簡單、通用、強大 —— 無論你是寫筆記、寫文件、寫 Prompt、寫部落格，Markdown 都是你最好的選擇。

更重要的是，AI 工具的輸入和輸出幾乎都基於 Markdown。學會 Markdown，等於學會了如何更有效地和 AI 溝通。 在這個人人都在使用 AI 的時代，掌握 Markdown 就是掌握了人機協作的基本功。
